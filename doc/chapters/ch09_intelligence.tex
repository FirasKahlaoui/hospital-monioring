\chapter{Statistical and Pre-trained Intelligence Layer}
\label{ch:intelligence}

\section{Methodological Disclaimer}
In accordance with engineering rigor, it is explicitly stated that the system does not utilize a custom-trained neural network for its intelligence layer. Instead, it leverages a hybrid approach combining \textbf{deterministic statistical modeling} for physiological telemetry and \textbf{pre-trained deep learning inference} for visual personnel identification. This strategy ensures computational efficiency on the service layer while maintaining high reliability for clinical monitoring.

\section{Telemetry Analysis: Statistical Modeling}
The intelligence layer formulates the monitoring task as a time-series analysis problem. The telemetry stream $V = \{v_1, v_2, ..., v_n\}$ is processed to detect acute physiological changes and project future states.

\subsection{Anomaly Detection via rolling Z-Score}
To identify clinical outliers without relying on static thresholds alone, the system implements a rolling Z-score analysis. For a window of size $k$ (where $5 \leq k \leq 20$), the mean $\mu$ and standard deviation $\sigma$ are computed. A data point $x$ is flagged as a statistical anomaly if its deviation from the mean exceeds a magnitude of 2.5:
\begin{equation}
    |Z| = \left| \frac{x - \mu}{\sigma} \right| > 2.5
    \label{eq:z_score_intel}
\end{equation}
This method allows the system to detect "shocks" to the patient's baseline vitals even if the values remain within globally "safe" ranges (e.g., a sudden jump from 60 to 90 BPM at rest).

\subsection{Short-Horizon Trend Projection}
Forecasting is implemented using \textbf{Ordinary Least Squares (OLS) Linear Regression}. The system estimates the slope $m$ of the vitals trend to project the subsequent 5 samples:
\begin{equation}
    m = \frac{n\sum(t_i v_i) - \sum t_i \sum v_i}{n\sum(t_i^2) - (\sum t_i)^2}
    \label{eq:ols_slope}
\end{equation}
The projection provides clinicians with a visual "direction of travel," enabling proactive intervention before critical thresholds are reached.

\section{Personnel Identification: Pre-trained Inference}
The personnel tracking system utilizes the \textbf{face-api.js} library, which provides high-level wrappers for pre-trained models including \textit{TinyFaceDetector} and \textit{FaceRecognitionNet}. 

\subsection{Face Descriptor Matching}
Identification is achieved by extracting a 128-dimensional descriptor vector $\mathbf{d} \in \mathbb{R}^{128}$ from the captured frame and calculating the \textbf{Euclidean distance} to enrolled staff signatures stored in Firestore. A match is confirmed if:
\begin{equation}
    \lVert \mathbf{d}_{detected} - \mathbf{d}_{enrolled} \rVert_2 < 0.4
\end{equation}
This threshold corresponds to a confidence level of $\approx 60\%$, chosen to balance sensitivity and false-positive reduction in varied hospital lighting conditions.

\section{Clinical Synthesis Engine}
The synthesized "AI Insight" is generated by a rule-based engine that combines the raw vitals, detected anomalies, and projected trends into a discrete clinical status:

\begin{itemize}
    \item \textbf{Stable:} Normal range, no anomalies, flat or improving trend.
    \item \textbf{Warning:} Approaching thresholds, persistent anomalies, or deteriorating trend.
    \item \textbf{Critical:} Threshold violation or high-magnitude instability.
\end{itemize}

\begin{algorithm}[H]
\caption{Intelligence Layer Synthesis (tRPC Procedure)}
\begin{algorithmic}[1]
\REQUIRE $V_{win}$ (Vitals history), $\mathbf{D}_{enrolled}$ (Staff descriptors)
\STATE $\mu, \sigma \gets \text{calcStats}(V_{win})$
\FOR{each $x \in V_{win}$}
    \IF{$|x - \mu| / \sigma > 2.5$} \STATE flagAnomaly($x$) \ENDIF
\ENDFOR
\STATE $m, b \gets \text{linearRegression}(V_{win})$
\STATE $V_{pred} \gets \{m(n+i) + b \mid i=1\dots5\}$
\STATE \textbf{Status Decision:}
\IF{$\text{latest}(V_{win}) \text{ violates thresholds}$}
    \STATE $S \gets \text{CRITICAL}$
\ELSIF{anomaliesDetected \OR $m > \text{threshold}$}
    \STATE $S \gets \text{WARNING}$
\ELSE
    \STATE $S \gets \text{STABLE}$
\ENDIF
\RETURN $S, V_{pred}, \text{insights}$
\end{algorithmic}
\end{algorithm}
